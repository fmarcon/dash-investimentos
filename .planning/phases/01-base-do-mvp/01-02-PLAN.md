## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → SPA | User input enters UI forms for investment data |
| SPA → backend API | JWT token and investment payloads cross the network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05 | S | frontend/app/login/page.tsx | mitigate | Store the JWT in memory or secure storage, not localStorage if possible, and clear it on logout. |
| T-06 | R | frontend/lib/api.ts | mitigate | Attach Authorization header only when JWT is present and avoid exposing token in logs. |
| T-07 | E | frontend/app/investments/page.tsx | mitigate | Validate required numeric fields in the form and render API errors clearly. |
| T-08 | I | frontend/app/investments/page.tsx | mitigate | Escape or sanitize any user-entered strings before rendering. |
</threat_model>
