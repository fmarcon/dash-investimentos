## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend → backend API | Untrusted user input enters the API on auth and investment endpoints |
| backend API → SQLite | Backend writes user-submitted investment data to local storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01 | S | backend/users/views.py | mitigate | Validate login/register payloads, hash passwords with Django, and never store plaintext passwords. |
| T-02 | I | backend/investments/views.py | mitigate | Enforce JWT auth and `filter(user=request.user)` on all investment endpoints. |
| T-03 | R | backend/investments/serializers.py | mitigate | Validate numeric fields and required dates before saving. |
| T-04 | S | backend/dashboard_investimentos/settings.py | mitigate | Configure CORS to allow only the SPA origin and disable debug mode for production. |
</threat_model>
