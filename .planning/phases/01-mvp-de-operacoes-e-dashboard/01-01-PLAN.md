<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→backend | User inputs and auth token flows from Next.js to Django API |
| frontend→cookie | JWT token stored in cookie HTTP-only to prevent script access |
| admin page | Admin-only data exposure from user list endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01 | S | backend/auth | mitigate | Use `HttpOnly`, `Secure`, and `SameSite` cookies for JWT and validate tokens server-side. |
| T-02 | I | frontend/admin | mitigate | Restrict admin endpoint to admin role and do not return password hashes. |
| T-03 | T | backend/api | mitigate | Validate all transaction payloads with DRF serializers and do not trust client-side role data. |
| T-04 | R | backend/api | mitigate | Implement input validation and return safe error messages; avoid exposing internal details. |
</threat_model>
